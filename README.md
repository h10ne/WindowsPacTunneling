# WindowsPacTunneling (WPT)

Windows-приложение для выборочного проксирования трафика: генерирует PAC-файл по спискам доменов и IP, поднимает локальный HTTP-сервер и включает системный PAC в Windows. Опционально запускает локальный прокси на базе [sing-box](https://github.com/SagerNet/sing-box) по ссылке (VLESS, VMess, Trojan, Shadowsocks).

## Как пользоваться

### Требования

- Windows 10/11
- [.NET 10 Runtime](https://dotnet.microsoft.com/download/dotnet/10.0) — только для запуска из исходников (`dotnet run`)
- [.NET 10 SDK](https://dotnet.microsoft.com/download/dotnet/10.0) — только для сборки из исходников

> Сборка из [Releases](https://github.com/h10ne/WindowsPacTunneling/releases/latest) — один `.exe` со встроенным runtime, отдельная установка .NET не нужна.

### Скачать

Готовую сборку можно взять на странице [Releases](https://github.com/h10ne/WindowsPacTunneling/releases/latest).


> Для изменения системных настроек прокси Windows может потребоваться запуск от имени администратора.

### Сценарий 1: PAC-туннелирование (основной режим)

PAC направляет в прокси только трафик к выбранным доменам и подсетям; остальное идёт напрямую.

1. Откройте вкладку **Тунелирование**.
2. Укажите адрес прокси (например `127.0.0.1:10808`) и порт локального PAC-сервера (по умолчанию `1080`).
3. Добавьте один или несколько **списков сервисов** (YouTube, Discord, Telegram и др.).
4. При необходимости добавьте **свои домены** или **IP/CIDR** в раскрывающихся блоках.
5. Нажмите **Применить** — приложение:
   - скачает или возьмёт из кэша списки доменов;
   - сгенерирует PAC-файл;
   - запустит локальный HTTP-сервер с PAC;
   - включит системный PAC в Windows.
6. Чтобы отключить — **Отключить PAC** на той же вкладке.
7. Кнопка **Показать PAC** открывает сгенерированный файл в Notepad.

Списки доменов берутся из репозитория [itdoginfo/allow-domains](https://github.com/itdoginfo/allow-domains) и кэшируются локально. Обновить вручную можно на вкладке **Настройки** → **Обновить списки**.

### Сценарий 2: Локальный прокси (sing-box)

Если у вас есть ссылка на внешний прокси, WPT может поднять локальный SOCKS/HTTP на `127.0.0.1` без изменения системных настроек.

1. Откройте вкладку **Прокси**.
2. Вставьте ссылку, например `vless://...`, `vmess://...`, `trojan://...` или `ss://...`.
3. Укажите локальный порт (по умолчанию `10808`).
4. Нажмите **Запустить** — при первом запуске sing-box скачается автоматически из GitHub.
5. На вкладке **Тунелирование** укажите `127.0.0.1:<локальный_порт>` как адрес прокси и примените PAC.

Локальный прокси **не меняет** системные настройки Windows сам по себе — его нужно связать с PAC вручную.

### Сценарий 3: Обход (YouTube, Discord, Telegram)

Режим для ситуаций, когда сервисы недоступны из‑за блокировок на уровне DPI, а не только по доменам. **Не использует PAC и не меняет системный прокси Windows** — работает параллельно с тунелированием и локальным прокси.

На вкладке **Обход** можно включить один или оба направления:

| Компонент | Что разблокирует | Как работает |
|---|---|---|
| **YouTube и Discord** | YouTube, Discord | [zapret](https://github.com/Flowseal/zapret-discord-youtube) — обход DPI через `winws.exe` и драйвер **WinDivert** |
| **Telegram** | Telegram Desktop | Локальный **MTProto WS-прокси** (реализация по мотивам [tg-ws-proxy](https://github.com/Flowseal/tg-ws-proxy)) на `127.0.0.1` |

#### Как пользоваться

1. Откройте вкладку **Обход**.
2. Отметьте **YouTube и Discord** и/или **Telegram**.
3. Для YouTube/Discord **запустите WPT от имени администратора** (нужен WinDivert). Включите это в **Настройки** → «Запускать от имени администратора» или кнопкой «Перезапустить от имени администратора».
4. Нажмите **Запустить обход**.
5. При первом запуске zapret **скачается автоматически** из GitHub (`Flowseal/zapret-discord-youtube`) в `%APPDATA%\WindowPacTunneling\zapret\`.
6. Если стратегия zapret ещё не сохранена, WPT **перебирает `general*.bat`**, проверяя доступ к YouTube и Discord, и запоминает рабочую.
7. Для Telegram после запуска в буфер копируется ссылка `tg://proxy?...` — в Telegram Desktop: **Настройки → Продвинутые → Тип соединения** → добавить MTProto-прокси.

Кнопка **Подобрать стратегию** вручную запускает перебор стратегий zapret (тоже требует прав администратора).

В **Расширенных настройках** на вкладке:
- порт MTProto-прокси (по умолчанию `1443`);
- текущая сохранённая стратегия zapret.

Обновить zapret можно в **Настройки** → секция **Zapret** → «Проверить обновления» / «Скачать обновление».

#### Что внутри (технически)

**Zapret**
- Устанавливается в `%APPDATA%\WindowPacTunneling\zapret\` (`winws.exe`, bat-стратегии `general*.bat`, бинарники в `bin\`).
- WPT запускает выбранный `.bat`, тот поднимает `winws.exe`.
- Проверка работоспособности: HTTP-запросы к `youtube.com/generate_204` и `discord.com/api/v9/experiments`.
- При обновлении останавливаются процессы zapret и при необходимости драйвер WinDivert.

**Telegram (TgWsProxy)**
- Встроенный в WPT локальный TCP-сервер на `127.0.0.1:<порт>`.
- Принимает MTProto от Telegram Desktop, проксирует через WebSocket к DC Telegram (с балансировкой Cloudflare-доменов).
- Секрет генерируется один раз и хранится в `tg-ws-proxy-secret.txt`; ссылка прокси имеет вид `tg://proxy?server=127.0.0.1&port=...&secret=dd...`.
- Для Telegram **права администратора не нужны**.

#### Ограничения

- YouTube/Discord **без прав администратора не запустятся** (WinDivert).
- Если ни одна стратегия zapret не сработала — проверьте интернет, DNS (в т.ч. Secure DNS в браузере) и попробуйте позже.
- Обход Telegram рассчитан на **Telegram Desktop** через MTProto-прокси, не на браузерную версию.

### Настройки и трей

На вкладке **Настройки** доступны:

- автозапуск вместе с Windows;
- запуск от имени администратора (нужен для обхода YouTube/Discord);
- автоматический запуск PAC при старте приложения;
- автоматический запуск обхода при старте (только при включённом запуске от администратора);
- сворачивание в системный трей при запуске;
- обновление списков при старте;
- обновление приложения и zapret;
- открытие папки данных приложения.

При закрытии окна приложение сворачивается в трей. Из контекстного меня иконки можно включить/отключить PAC и управлять локальным прокси.

### Папка данных

Все пользовательские данные хранятся в:

```
%APPDATA%\WindowPacTunneling\
```

| Файл / папка | Назначение |
|---|---|
| `settings.json` | Настройки приложения |
| `proxy.pac` | Последний сгенерированный PAC |
| `lists\` | Кэш списков доменов и подсетей |
| `bin\sing-box.exe` | Локальный прокси-клиент |
| `sing-box.json` | Конфигурация sing-box |
| `zapret\` | Zapret: `winws.exe`, стратегии, `version.txt` |
| `tg-ws-proxy-secret.txt` | Секрет локального MTProto-прокси для Telegram |
